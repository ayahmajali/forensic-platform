Download the ForensicPlatform Agent

What the agent scans

The agent walks the folder you point it at, hashes every file (MD5 + SHA-256), records MAC timestamps, classifies files by type, and performs type-specific extraction:

  Images
EXIF via exiftool — GPS, camera make/model, serial number, timestamps.
JPGJPEGBMPPNGTIFF
  Documents
Text preview, page count, authorship metadata.
PDFDOCXTXTCSVLOGMD
  Archives
Member list, CRC, compressed/uncompressed sizes. RAR is opt-in (prompts you).
ZIPRAR
  Disk Images
Flagged for optional upload — Sleuth Kit runs server-side for partitioning + recovery.
E01DDRAWIMGISO.001VMDK
Privacy: for logical files, only the metadata — hashes, EXIF, text previews (first ~1500 chars), archive listings — is sent to the backend. Your raw photos, documents and archives stay on your computer. Disk images are only uploaded when you explicitly pass --upload-images.
Install & run the agent
  1. Download the macOS binary above and make it executable: 
    chmod +x ~/Downloads/forensic-agent-macos
sudo mv ~/Downloads/forensic-agent-macos /usr/local/bin/forensic-agent
  2. Configure the connection — set the backend URL and your agent key: 
    export FORENSIC_API_URL="http://localhost:8000"
export FORENSIC_API_KEY="<paste the AGENT_API_KEY your admin gave you>"
    Add those two lines to ~/.zshrc so they persist across terminals.
  3. Scan a folder locally — nothing leaves your machine except the findings JSON: 
    forensic-agent scan ~/Documents/SuspiciousFolder --watch
# open the case in your browser when done:
open $FORENSIC_API_URL/results/<CASE_ID>
  4. Uploading a disk image (E01, DD, RAW…) for Sleuth Kit: 
    forensic-agent upload ~/Evidence/disk.e01 --watch --download-report ./case.pdf
First launch may show “cannot be opened because the developer cannot be verified”. Run xattr -dr com.apple.quarantine /usr/local/bin/forensic-agent to clear the quarantine flag, or right-click → Open once.
  1. Download forensic-agent-windows.exe and drop it in a folder on your PATH (e.g. C:\Tools\).
  2. Configure the connection in PowerShell: 
    $env:FORENSIC_API_URL = "http://localhost:8000"
$env:FORENSIC_API_KEY = "<paste the AGENT_API_KEY your admin gave you>"
    To persist them permanently, open Edit environment variables for your account from the Start menu and add them there.
  3. Scan a folder locally: 
    C:\Tools\forensic-agent-windows.exe scan C:\Users\You\Documents\SuspiciousFolder --watch
  4. Uploading a disk image: 
    C:\Tools\forensic-agent-windows.exe upload D:\Evidence\disk.e01 --watch --download-report .\case.pdf
Windows SmartScreen may flag the download the first time. Click More info → Run anyway. The binary is unsigned because this is an academic project — inspect the source if you want to build it yourself.

If you prefer not to trust a pre-built binary, install the agent from source — it's pure Python and works on macOS, Linux and Windows.

  1. Clone the repo: 
    git clone https://github.com/ayahmajali/forensic-platform.git
cd forensic-platform/agent
  2. Install into a virtual environment: 
    python3 -m venv .venv-agent
source .venv-agent/bin/activate  # Windows: .venv-agent\Scripts\Activate.ps1
pip install -e .
  3. Run the scan — same commands as the binary: 
    forensic-agent scan /path/to/folder --watch --download-report ./case.pdf
  4. Optional extras — richer parsing: 
    # PDF text extraction
pip install pypdf

# DOCX text extraction
pip install python-docx

# Image EXIF (exiftool binary, not pip)
brew install exiftool      # macOS
choco install exiftool    # Windows w/ Chocolatey

# RAR archive reading (opt-in; agent will prompt)
pip install rarfile
brew install rar          # macOS
CLI reference
# Health check
forensic-agent health

# Local scan (recommended for folders)
forensic-agent scan <path> [--include-rar|--skip-rar] [--upload-images]
                          [--watch] [--download-report ./report.pdf]
                          [--output-json ./findings.json]

# Full upload (for disk images)
forensic-agent upload <path> [--keywords a,b,c] [--investigator "Jane Doe"]
                            [--watch] [--download-report ./report.pdf]

# Poll a case
forensic-agent status <CASE_ID>
forensic-agent watch  <CASE_ID>

# Download PDF for an existing case
forensic-agent report <CASE_ID> -o ./case.pdf
Where do I get an API key? Your admin generates one on the server with python -c "import secrets; print(secrets.token_urlsafe(32))" and sets AGENT_API_KEY in backend/.env. Every agent user is given the same key (or a comma-separated rotating list) and puts it in FORENSIC_API_KEY.
