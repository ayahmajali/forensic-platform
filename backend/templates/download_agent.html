Download the ForensicPlatform Agent

A native desktop application that scans a folder or disk image on your computer, hashes every file, extracts metadata, and sends a compact findings package to this platform for reporting. What the agent scans

The agent walks the folder you point it at, hashes every file (MD5 + SHA-256), records MAC timestamps, classifies files by type, and performs type-specific extraction:

  Images
EXIF via exiftool — GPS, camera make/model, serial number, timestamps.
JPGJPEGBMPPNGTIFF
  Documents
Text preview, page count, authorship metadata.
PDFDOCXTXTCSVLOGMD
  Archives
Member list, CRC, compressed/uncompressed sizes. RAR is opt-in (prompts you).
ZIPRAR
  Disk Images
Flagged for optional upload — Sleuth Kit runs server-side for partitioning + recovery.
E01DDRAWIMGISO.001VMDK
Privacy: for logical files, only the metadata — hashes, EXIF, text previews (first ~1500 chars), archive listings — is sent to the backend. Your raw photos, documents and archives stay on your computer. Disk images are only uploaded when you explicitly pass --upload-images.
Install & run the agent
  1. Download ForensicAgent-macos.zip using the button above, then double-click the zip in Finder to extract ForensicAgent.app.
  2. Drag ForensicAgent.app into your Applications folder (or anywhere you like — this is just so Spotlight can find it later).
  3. Launch the app. Because the app isn't signed with a paid $99/year Apple Developer certificate, macOS Gatekeeper will block the first launch with one of these dialogs:
    • "ForensicAgent cannot be opened because the developer cannot be verified"
    • "macOS cannot verify that this app is free of malware"
    Fix it once with this one-liner in Terminal — then double-click and it'll open every time afterwards: 
    # Adjust the path if you put the app somewhere other than /Applications
xattr -cr /Applications/ForensicAgent.app
codesign --force --deep --sign - /Applications/ForensicAgent.app
    xattr -cr strips the com.apple.quarantine flag Safari/Chrome add to every downloaded file. codesign --sign - applies an ad-hoc signature — not a real Apple one, but enough to satisfy Gatekeeper's integrity check. Neither of these requires admin privileges; a forensic scanner shouldn't need them.
  4. Fill in the connection fields — the app pre-fills the backend URL with https://forensic-platform-sy5q.onrender.com. Paste the Agent API Key your admin gave you into the API Key field and click Test Connection.
  5. Pick a folder to scan with the Choose Folder… button, then hit ▶ Start Scan. Watch the progress bar; if the folder contains .rar archives you'll get a dialog asking whether to include them.
  6. Submit & review — when the scan finishes you'll see a file-type breakdown. Click Submit to Backend and your Case ID appears. Click 🌐 Open Case in Browser to see the full report on this website.
Nothing inside the scanned folder is ever uploaded — only the JSON findings (hashes, metadata, file list). Disk images like .e01 / .dd are the one exception and are flagged in the summary so you can decide whether to upload them separately for Sleuth Kit processing.
  1. Download ForensicAgent-windows.exe using the button above.
  2. Double-click the .exe. If Windows SmartScreen warns "Windows protected your PC", click More info → Run anyway. The binary is unsigned because this is an academic project — you can inspect or rebuild it from source any time.
    A native window titled Forensic Agent 1.0.0 will open. No terminal, no PowerShell, no environment variables.
  3. Fill in the connection fields — the app pre-fills the backend URL with https://forensic-platform-sy5q.onrender.com. Paste the Agent API Key into the API Key field and click Test Connection.
  4. Pick a folder with Choose Folder… (for example C:\Users\You\Documents\SuspiciousFolder), then click ▶ Start Scan.
  5. Submit & review — when the scan finishes, click Submit to Backend. Your Case ID appears with a 🌐 Open Case in Browser button that takes you straight to the report.
The app never uploads raw file contents — it only POSTs a JSON findings document to /api/agent/findings. Disk images (.e01, .dd, .raw, …) are flagged in the summary for optional server-side processing.

If you prefer not to trust a pre-built binary, install the agent from source — it's pure Python and works on macOS, Linux and Windows.

  1. Clone the repo: 
    git clone https://github.com/ayahmajali/forensic-platform.git
cd forensic-platform/agent
  2. Install into a virtual environment: 
    python3 -m venv .venv-agent
source .venv-agent/bin/activate  # Windows: .venv-agent\Scripts\Activate.ps1
pip install -r requirements.txt
  3. Launch the GUI — exactly what the downloadable binary does, just from source: 
    python forensic_agent_gui.py
  4. Or use the CLI — useful for automation, CI pipelines, or when scanning over SSH: 
    python forensic_agent.py scan /path/to/folder --watch --download-report ./case.pdf
  5. Build your own .app / .exe: 
    # macOS
chmod +x build_macos.sh && ./build_macos.sh
# Windows (PowerShell or cmd)
.\build_windows.bat
  6. Optional extras — richer parsing (install these alongside requirements.txt and the scanner picks them up automatically): 
    # Image EXIF needs the exiftool binary, not pip
brew install exiftool      # macOS
choco install exiftool    # Windows w/ Chocolatey

# RAR archive reading (the scanner will prompt)
brew install rar          # macOS
# Windows: install WinRAR from rarlab.com
CLI reference — power users only; the desktop app does all of this with clicks
# Health check
forensic-agent health

# Local scan (recommended for folders)
forensic-agent scan <path> [--include-rar|--skip-rar] [--upload-images]
                          [--watch] [--download-report ./report.pdf]
                          [--output-json ./findings.json]

# Full upload (for disk images)
forensic-agent upload <path> [--keywords a,b,c] [--investigator "Jane Doe"]
                            [--watch] [--download-report ./report.pdf]

# Poll a case
forensic-agent status <CASE_ID>
forensic-agent watch  <CASE_ID>

# Download PDF for an existing case
forensic-agent report <CASE_ID> -o ./case.pdf
Where do I get an API key? Your admin generates one on the server with python -c "import secrets; print(secrets.token_urlsafe(32))" and sets AGENT_API_KEY in backend/.env. Every agent user is given the same key (or a comma-separated rotating list) and puts it in FORENSIC_API_KEY.
